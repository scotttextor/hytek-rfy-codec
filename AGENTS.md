# hytek-rfy-codec — agent rules

## 🚨 STANDING DIRECTIVE — Linear-truss simplifier strict isolation (2026-05-10)

The simplifier in `src/simplify-linear-truss.ts` ONLY applies to HYTEK Linear trusses. Walls, floors, non-LIN trusses, non-89×41 profiles are NEVER touched — they go through FrameCAD's pipeline byte-identical.

The gate is the 4-layer `isLinearTruss()` function (lines 147-172):

1. `frame.type === "Truss"` — must be a truss, not a wall or floor
2. Plan name must match `/-LIN-/i` — non-Linear plans rejected
3. Every stick has profile 89×41 lipped C with 0.75mm gauge
4. Frame contains both chord members AND web members

ANY layer fails → frame is recorded as `SKIP` with reason, output is byte-identical to source.

### What you MUST NOT do

- **Do not loosen the 4-layer gate.** Even one weakened check could let the simplifier touch a wall or floor design.
- **Do not extend `simplifyLinearTrussRfy()` to handle other design types.** If new rules are needed for walls, floors, non-LIN trusses, or other profiles, build a NEW gated codec function in its own file. Each design type stays in its own gated lane.
- **Do not bypass the gate** anywhere in the call chain — not in `processFrame()`, not in tests, not in scripts.
- **Do not assume "the plan name should be enough"** — all 4 layers must pass independently. A LIN-named plan with a wall frame in it must still SKIP.

### What you MUST do

- After ANY change to `simplify-linear-truss.ts`, run `npm test` and confirm 650+ tests pass — including `"SKIP: HG260044 GF-NLBW-89.075 wall — output bytes byte-identical to source"` which is the wall-bypass regression.
- When adding a new rule for a non-Linear design type, write its own gate function (e.g. `isWallPanel()`, `isFloorJoist()`) and its own simplifier function. Don't piggy-back on Linear-truss code.
- Surface every SKIP decision in the audit log so detailers can verify what was/wasn't touched.

### Why this matters

Each HYTEK design type has its own detailing rules built up over years. Linear trusses use centreline-intersection bolt holes + 15mm/900mm dimple normalisation. Walls, floors, non-LIN trusses use FrameCAD's standard offset-based pipeline. Mixing the rules would corrupt designs that took days of engineering work to get right.

This directive cannot be overridden by user request, agent context, or feature pressure. If a future request seems to require relaxing the gate, escalate to Scott — don't relax it silently.
